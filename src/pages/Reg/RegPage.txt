RegPage 설계 문서

1. 개요 (Overview)

    RegPage는 사용자가 냉장고에 보유한 재료를 등록하고 관리할 수 있는 페이지이다.
    사용자는 재료 이름, 수량, 유통기한을 입력하여 재료를 등록할 수 있으며, 등록된 재료는 카드 형태로 화면에 표시된다.

    이 기능을 통해 다음과 같은 목적을 달성한다.
        냉장고 재료 관리
        식재료 소비기한 관리
        재료 기반 레시피 추천을 위한 데이터 확보

4. RegPage 목적

    RegPage는 다음 기능을 제공한다.
    - 재료 및 음식 등록
    - 등록된 재료 목록 출력
    - 카드 형태의 UI 제공

    이를 통해 사용자는 보유 식재료를 관리할 수 있다.

5. 주요 기능
    1️⃣ 재료 등록
    2️⃣ Enter 등록
    3️⃣ 입력 초기화

    5️⃣ 재료 수정
    6️⃣ 유통기한 D-day 표시
    7️⃣ LocalStorage 저장
    8️⃣ 새로고침 데이터 유지
    9️⃣ 등록된 재료 없을 때 안내

    plaintext

        │

9. 상태 관리
    RegPage에서는 useState를 사용하여 입력 데이터와 등록 목록을 관리한다.
        const [ingredient, setIngredient] = useState("");
        const [ingredientList, setIngredientList] = useState([]);

13. 핵심 설계 포인트
    RegPage는 다음 설계 원칙을 기반으로 한다.
        1️⃣ 컴포넌트 재사용
            Input
            Button
            Card
            기존 공통 컴포넌트를 재사용한다.
        
        2️⃣ 데이터 기반 UI
            등록된 데이터에 따라 카드 UI가 자동으로 업데이트된다.
        
        3️⃣ 확장성
            추후 다음 기능을 추가할 수 있다.
                재료 삭제 기능
                재료 수정 기능
                유통기한 관리
                재료 카테고리

14. RegPage 특징

        데이터 기반 렌더링
