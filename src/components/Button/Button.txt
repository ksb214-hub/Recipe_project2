Button 페이지 설계 문서
1. 개요 (Overview)
    Button 컴포넌트는 zero-naeng-fe 프로젝트에서 반복적으로 사용되는 입력 UI를 재사용하기 위해 제작된 공통 컴포넌트이다.

    여러 페이지에서 동일한 Button 구조를 사용함으로써 다음과 같은 문제를 해결할 수 있다.
        - 반복되는 input 코드 작성 방지
        - UI 디자인 통일
        - 유지보수 효율 향상


2. 개발 환경
    항목		내용
	Framework	React
	Language	JavaScript
	Routing		React Router
	Component	Input, Button
	Styling	    CSS


3. 폴더 구조
    src/
    ├─ components/
    │   └─ Button/
    │       ├─ Button.js
    │       └─ Button.css

     구성 요소
	    -> Button.js → Button 컴포넌트 구조 및 기능
	    -> Button.css → Button 스타일 정의

4. Button 컴포넌트 목적
    Button 컴포넌트는 다음과 같은 목적을 가진다.
        -> 공통 입력 UI 제공
        -> 다양한 페이지에서 재사용 가능
        -> props 기반 입력 처리
        -> React 상태 관리와 연결

5. Button 주요 기능
    Button 기능
    │
    ├── 버튼 UI 제공
    │
    ├── props 기반 기능
    │     ├── type
    │     ├── variant
    │     ├── disabled
    │     └── children
    │
    ├── 이벤트 처리
    │     └── onClick
    │
    └── 스타일 확장
        ├── primary
        ├── secondary
        ├── outline
        └── disabled

6. Button 컴포넌트 구조
    Button
    │
    └── button.btn
        │
        ├── primary
        ├── secondary
        └── outline

7. Button 스타일 구조
    Button 스타일은 기본 스타일 + variant 스타일로 구성된다.
    
        btn (기본 스타일)
        │
        ├── primary
        │
        ├── secondary
        │
        └── outline

8. props 구조
    Button 컴포넌트는 다음 props를 사용한다.
        props           설명
        type        버튼 타입 (button, submit)
        variant	    버튼 스타일 (primary, secondary, outline)
        disabled	버튼 비활성화
        children	버튼 내부 텍스트

9. Button 상태
    Button 컴포넌트는 다음 상태를 지원한다.
        -> 기본 상태
        -> hover 상태
        -> disabled 상태

    예시
        .btn.primary {
            background-color: var(--color-primary);
        }

        .btn.primary:hover {
            background-color: var(--color-primary-hover);
        }

        .btn:disabled {
            background-color: #ccc;
        }
10. 데이터 흐름
    Button 컴포넌트는 이벤트 기반 구조로 동작한다.

        사용자 클릭
            │
            ▼
        Button 클릭 이벤트
            │
            ▼
        onClick 실행
            │
            ▼
        부모 컴포넌트 로직 실행

11. Button 컴포넌트 특징
    -> 재사용 가능한 UI
    -> 컴포넌트 기반 설계
    -> props 기반 스타일 제어
    -> 이벤트 처리 가능
    -> 확장 가능한 구조

12. 정리
    Button 컴포넌트는 프로젝트에서 버튼 UI를 표준화하기 위해 설계된 공통 컴포넌트이다.
    이를 통해 다음과 같은 효과를 얻을 수 있다.
        -> 코드 중복 감소
        -> UI 일관성 유지
        -> 유지보수 효율 향상
        -> 확장성 확보
