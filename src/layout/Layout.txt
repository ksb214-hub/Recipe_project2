Layout 컴포넌트 설계 문서

1. 개요 (Overview)
    Layout 컴포넌트는 zero-naeng-fe 프로젝트의 공통 레이아웃을 관리하는 컴포넌트이다.
    프로젝트의 모든 페이지에서 동일하게 사용되는 UI 요소를 하나의 구조로 통합하기 위해 설계되었다.

    예를 들어 다음 요소들을 관리한다.
        Header (네비게이션)
        main 영역
        공통 페이지 구조

2. Layout 컴포넌트 목적
	공통 UI 중앙 관리
	코드 중복 제거
	유지보수 효율 증가


3. 폴더 구조
    src/
    │
    ├─ layout/
    │   └─ Layout.js

    구성 요소
		Layout.js → 공통 레이아웃 관리

4. Layout 사용 구조
    Layout 컴포넌트는 App.js에서 페이지를 감싸는 구조로 사용된다. 

    App.js 구조

        import Layout from "./layout/Layout";
        <Layout>
            <Routes>
                <Route path="/" element={<Main />} />
                <Route path="/login" element={<Login />} />
                <Route path="/search" element={<SearchPage />} />
            </Routes>
        </Layout>

5. Layout 구조    
    Layout
    │
    ├── Header
    │
    └── main
        └── children (페이지)

6. Layout 코드 구조
    import Header from "../components/Header/Header";
    function Layout({ children }) {
    return (
        <>
        <Header />

        <main style={{ marginTop: "60px" }}>
            {children}
        </main>
        </>
    );
    }
    export default Layout;

7. children 개념
    Layout 컴포넌트에서 children은 내부에 들어오는 페이지를 의미한다.   
    즉 Layout은 페이지를 감싸는 Wrapper 역할을 한다.

8. 구조 흐름 (Component Flow)
    App
    │
    ├── Layout
    │     │
    │     ├── Header
    │     │
    │     └── main
    │           │
    │           └── Page
    │                ├── Main
    │                ├── Login
    │                └── Search

10. Layout 특징
    Layout 컴포넌트는 다음 특징을 가진다.
        공통 UI 관리
        페이지 구조 통일
        Header 중앙 관리
        React Router와 연결

11. 중요한 UI 문제 해결
    Header는 position: fixed로 설정되어 있다.
    이 경우 페이지 내용이 Header와 겹칠 수 있다.

    해결 방법
        main {
            margin-top: 60px;
        }
    이렇게 하면 Header와 main 영역이 겹치지 않는다.

12. Layout 설계 핵심 포인트
    1️⃣ 공통 UI 분리
        Header를 모든 페이지에서 공통 사용
    2️⃣ Wrapper 구조
        Layout은 페이지를 감싸는 구조
    3️⃣ 유지보수성
        Header 수정 시
            Header.js만 수정
        모든 페이지 자동 적용
    
    4️⃣ 확장성
        Layout에 다음 기능 추가 가능
            Footer
            Sidebar
            Dark mode
            Loading UI
13. 정리
    Layout 컴포넌트는 프로젝트 전체 페이지 구조를 관리하는 핵심 컴포넌트이다.
    이를 통해 다음 효과를 얻을 수 있다.
        공통 UI 관리
        코드 중복 제거
        유지보수 효율 향상
        확장성 확보
